# PLF-026: Add ticket workflow doctor command

## Objective

Add a read-only audit command that checks ticket workflow consistency and reports actionable errors and warnings.

## Context

As ticket workflow commands and advisory artifacts accumulate, occasional consistency checks will help preserve the file-based workflow without adding hooks or a heavy project-management system.

## Scope

In scope:

- Add a command such as `/ticket-doctor`.
- Keep the command deterministic and display-only by default; do not trigger parent LLM handoff unless a later explicit option is approved.
- Detect duplicate ticket IDs across ticket state directories.
- Detect multiple ongoing tickets.
- Detect missing expected ticket directories.
- Detect malformed planned or ongoing tickets when required sections are absent.
- Detect stale advisory artifacts when artifact conventions support this.
- Detect artifacts whose source ticket no longer exists.
- Report errors and warnings separately.

Out of scope:

- Auto-repair.
- Moving tickets.
- Deleting stale artifacts.
- Git hooks.
- Replacing the manual workflow.

## Acceptance Criteria

- [x] The command is read-only.
- [x] It reports duplicate IDs and multiple ongoing tickets as errors.
- [x] It reports missing required sections in planned or ongoing tickets as warnings or errors according to documented policy.
- [x] It reports stale or orphaned advisory artifacts when detectable.
- [x] It does not treat artifacts as authoritative workflow state.
- [x] It provides actionable next steps without applying fixes.
- [x] It does not trigger parent LLM handoff by default.

## Dependencies

- `PLF-019` — advisory artifact conventions are needed before stale artifact checks are meaningful.

## Implementation Notes

Likely files:

- `.pi/extensions/ticket-workflow/index.ts`
- `docs/ticket-workflow-commands.md`

Minimal checks:

- expected ticket state directories exist;
- every `PLF-*.md` ticket ID appears in at most one state directory;
- `tickets/ongoing/` contains at most one `PLF-*.md` file;
- planned and ongoing tickets contain the required workflow sections from `tickets/TEMPLATE.md`;
- advisory artifacts under `tickets/.artifacts/<type>/<id>.md` are checked for missing source tickets, mismatched recorded path, mismatched recorded state, and mismatched SHA-256 when metadata is present.

This command should run locally and deterministically. It should not require child Pi analysis.

## Decisions

- Implemented `/ticket-doctor` as a deterministic display-only command, not an advisory command, so it does not use parent LLM handoff.
- Classified duplicate ticket IDs, multiple ongoing tickets, missing ticket state directories, and unreadable state directories as errors.
- Classified malformed planned or ongoing tickets and advisory artifact inconsistencies as warnings because ticket files and state directories remain authoritative.
- Reused the existing advisory artifact metadata fields and SHA-256 helper to detect stale artifacts without treating artifacts as workflow state.
- Kept the command read-only and did not add auto-repair behavior.

## Notes

Lower priority until advisory artifacts exist and the workflow command set starts being used.

Unlike advisory brief commands, this deterministic audit command should remain display-only by default. A future explicit option can add parent LLM handoff if repeated usage shows that a synthesized explanation is valuable.
