# PLF-025: Add ongoing ticket completion brief command

## Objective

Add a helper command that prepares a completion checklist for the single ongoing ticket without committing or moving the ticket.

## Context

Ticket completion has several required checks: acceptance criteria, verification, changed files, decisions, commit message, and final ticket movement. A completion brief can make this reliable while keeping commit and state transition under human control.

## Scope

In scope:

- Add a command such as `/ticket-completion-brief`.
- Require exactly one ongoing ticket.
- Read the latest verification brief and warn if missing, stale, failed, or inconclusive.
- Summarize changed files.
- List missing completion requirements from `tickets/README.md`.
- Suggest a commit message using the ticket ID.
- Optionally write an advisory completion artifact if conventions include one.
- Hand the completion recommendation back to the parent LLM according to the `PLF-029` convention.

Out of scope:

- Running `git commit`.
- Moving the ticket to `completed/`.
- Editing the ticket resolution automatically.
- Deciding that completion is allowed without human approval.

## Acceptance Criteria

- [x] The command refuses to run when there is zero or more than one ongoing ticket.
- [x] The output includes changed files, verification status, and a suggested commit message.
- [x] It warns when verification is missing, stale, failed, or inconclusive.
- [x] It lists remaining completion checklist items.
- [x] The command hands the completion recommendation back to the parent LLM or prints the suggested handoff in print mode.
- [x] It does not commit or move files.

## Decisions

- Implemented `/ticket-completion-brief` as a read-only advisory command requiring exactly one `PLF-*.md` file in `tickets/ongoing/`, mirroring the `/ticket-verify` handler flow.
- Reused existing helpers `requireSingleOngoingTicket`, `inspectGitChangedFiles`, `buildAdvisoryArtifactPath`, `buildAdvisoryArtifactMetadata`, and `deliverAdvisoryParentHandoff`.
- Added `inspectVerificationArtifact` analogous to `inspectPlanArtifact`, with best-effort verdict parsing (`pass`/`fail`/`inconclusive`) and SHA-256 staleness detection.
- Added a deterministic parent-side completion checklist builder derived from the `ongoing -> completed` requirements in `tickets/README.md`, marking items as `satisfied`, `pending`, or `human-to-confirm` without fabricating satisfaction.
- Added `readTicketTitle` (frontmatter then first heading) and `buildSuggestedCommitMessage` to produce the `PLF-NNN: <title>` form.
- Used an inline prompt helper rather than adding a separate prompt file to keep the MVP implementation small.
