# PLF-027: Add TypeScript editor support for Pi extensions

## Objective

Add portable local TypeScript editor and type-checking support for project-local Pi extensions without adding a root-level Node project.

## Context

VS Code showed TypeScript diagnostics in project-local extension files because the editor could not resolve Pi extension types, Node types, or extension dependencies from the repository. A temporary absolute-path `tsconfig.json` fixed one extension but depended on a local Homebrew installation path. The durable fix should keep the setup portable and scoped to `.pi/extensions/`.

## Scope

In scope:

- Add a `.pi/extensions/` TypeScript project configuration for editor support and local type-checking.
- Add `.pi/extensions/package.json` and lockfile with the development dependencies needed to resolve extension imports.
- Keep `node_modules/` untracked.
- Include the existing project-local extensions in the TypeScript configuration.
- Fix small TypeScript incompatibilities discovered by the type checker when they are directly related to current Pi extension API types.
- Verify the configuration with a local TypeScript no-emit check.
- Verify an existing Pi command still loads.

Out of scope:

- Adding a root-level `package.json`.
- Reworking extension architecture.
- Adding tests beyond the no-emit type check.
- Changing runtime behavior except for minimal API-compatible typing fixes.
- Implementing child Pi advisory runner behavior from `PLF-020`.

## Acceptance Criteria

- [x] VS Code can resolve TypeScript diagnostics for all current `.pi/extensions/**/*.ts` files using portable repository-local configuration.
- [x] The TypeScript configuration does not contain machine-specific absolute paths.
- [x] `npm --prefix .pi/extensions exec tsc -- -p .pi/extensions/tsconfig.json --noEmit` passes.
- [x] `/ticket-status` still runs successfully.
- [x] The implementation does not add a root-level Node project.
