# PLF-010: Create reusable source cards

Backlog tickets may be incomplete. Before moving this ticket to `planned`, refine whether this should be prompt-only, wiki-only, or tool-assisted.

## Objective

Create normalized source cards for important LinuxFr sources so agents can reuse evidence without reinterpreting raw pages from scratch.

## Context

The V2 roadmap lists source cards as a hybrid local-structure feature. The expanded radar report repeatedly identified central sources such as DarkMoon, ELY, LiberClaw, and the agent pricing journal.

Motivating sources:

- `docs/linuxfr-v2-system-roadmap.md`
- `docs/prototype-prompts/agentic-ai-career-radar-expanded/review.md`
- `docs/prototype-prompts/agentic-ai-career-radar-expanded/result.md`

## Scope

In scope:

- Define a small source card template with URL, local path, source type, title, date, topic tags, key claims, relevance, caveats, and citations.
- Generate cards for selected high-value sources only.
- Link cards from wiki notes or reports.

Out of scope:

- Card generation for every raw source by default.
- Complex taxonomy or ontology.
- Replacing raw sources.
- Vector indexing.

## Acceptance Criteria

- [ ] To be refined before planning.
- [ ] A card includes enough metadata and cited claims for reuse.
- [ ] Cards clearly distinguish source claims from agent interpretation.
- [ ] The first version remains lightweight Markdown.
