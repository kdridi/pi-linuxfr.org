# PLF-003: Parse tag and listing seed pages

Backlog tickets may be incomplete. Before moving this ticket to `planned`, refine the scope and verification method.

## Objective

Extract structured LinuxFr detail links and lightweight metadata from already collected tag pages and listing pages.

## Context

The expanded Agentic AI Career Radar run relied heavily on tag and listing pages as discovery surfaces, but link extraction was manual and brittle. The V2 roadmap calls out tag/listing parsing as a discovery primitive.

Motivating sources:

- `docs/linuxfr-v2-system-roadmap.md`
- `docs/prototype-prompts/agentic-ai-career-radar-expanded/review.md`
- `docs/prototype-prompts/agentic-ai-career-radar-expanded/result.md`

## Scope

In scope:

- Parse local HTML pages collected from LinuxFr tag URLs and top-level listings.
- Extract internal detail URLs.
- Classify obvious target types such as news, journal, link, forum, and poll.
- Preserve source seed path, source URL, anchor text, and nearby context when possible.

Out of scope:

- Crawling extracted URLs.
- Following pagination automatically.
- Parsing comments.
- Scoring candidates beyond basic extraction metadata.

## Acceptance Criteria

- [ ] To be refined before planning.
- [ ] At least one collected tag page and one listing page can produce an inspectable list of LinuxFr detail URLs.
- [ ] Extracted URLs are local-source-derived and do not trigger network access.
- [ ] Output is suitable as input to candidate ranking or manual selection.
