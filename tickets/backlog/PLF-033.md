# PLF-033: Add trusted backlog planning contract applier

## Objective

Add a trusted command that applies a human-approved backlog planning contract without redoing the strategic analysis that produced the contract.

## Context

After `PLF-032`, the system can analyze selected backlog tickets once and produce a contract describing proposed edits, splits, dependency additions, new tickets, and moves to `planned/`. The user wants to approve that result, then let automation apply exactly that approved contract. Re-running the same strategic analysis during apply would be wasteful and could produce a different LLM conclusion, so apply must consume the approved contract as its source.

## Scope

In scope:

- Add a command such as `/ticket-backlog-contract-apply <contract-path-or-id> --trusted`.
- Require an explicit trusted/approved invocation.
- Refuse to start with a dirty working tree unless the contract explicitly allows a known artifact-only state.
- Run deterministic status and doctor checks before mutation.
- Verify every source fingerprint recorded in the contract before applying changes.
- Apply only the edits, ticket creations, dependency changes, splits, and moves recorded in the contract.
- Add ticket log entries for automated changes and transitions.
- Stop without partial hidden repair when contract inputs are stale or expected files do not match.
- Report exactly what was applied and what remains for the human.

Out of scope:

- Re-analyzing tickets strategically.
- Inventing new splits, dependencies, or edits not present in the approved contract.
- Implementing planned tickets.
- Committing automatically unless explicitly accepted by a later ticket.
- Repairing workflow corruption.

## Acceptance Criteria

- [ ] The command refuses unapproved or non-trusted apply attempts.
- [ ] The command refuses stale contracts when source ticket fingerprints no longer match.
- [ ] The command applies only contract-listed changes.
- [ ] The command records log entries in changed tickets.
- [ ] The command moves approved tickets to `tickets/planned/` when the contract says to do so.
- [ ] The command stops clearly on unexpected state and reports the next human action.
- [ ] The command does not perform implementation work or commits.

## Dependencies
