# PLF-007: Collect public comments for selected pages

Backlog tickets may be incomplete. Before moving this ticket to `planned`, refine the scope, safety constraints, and verification method.

## Objective

Add a bounded way to collect public LinuxFr comments for explicitly selected detail pages.

## Context

The V2 roadmap and expanded radar review both identify comments as LinuxFr's main discussion value. The current MVP misses objections, corrections, consensus, and community knowledge because it collects page bodies but not structured discussions.

Motivating sources:

- `docs/linuxfr-v2-system-roadmap.md`
- `docs/prototype-prompts/agentic-ai-career-radar-expanded/review.md`
- `docs/use-case-challenge-prompts/bold-use-cases/response-grok.md`
- `docs/use-case-challenge-prompts/disruptive-cultural-intelligence/prompt.md`

## Scope

In scope:

- Collect public anonymous comments only for explicit LinuxFr detail URLs selected by a human or agent.
- Keep comment collection bounded by page count and possibly comment count.
- Store raw comment source material locally with source URLs and metadata.

Out of scope:

- Authentication.
- Posting, voting, or interacting with comments.
- Bulk comment crawling.
- Author profiling.
- Comment synthesis or argument extraction.

## Acceptance Criteria

- [ ] To be refined before planning.
- [ ] Comment collection is explicitly URL-driven and bounded.
- [ ] Raw collected comments cite source URL and local path.
- [ ] The implementation respects the public-read-only website access policy.
