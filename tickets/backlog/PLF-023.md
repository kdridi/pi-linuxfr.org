# PLF-023: Add ongoing ticket verification brief command

## Objective

Add a read-only advisory command that verifies the single ongoing ticket against its acceptance criteria, implementation plan, and current repository changes.

## Context

Completing a ticket safely requires checking scope, acceptance criteria, changed files, and verification evidence. A child Pi can perform this review in a separate context and produce a verification brief, while the orchestrator and human decide whether completion is appropriate.

## Scope

In scope:

- Add a command such as `/ticket-verify` for the current ongoing ticket.
- Require exactly one `PLF-*.md` file in `tickets/ongoing/`.
- Read the ongoing ticket.
- Read the latest non-stale implementation plan artifact when present.
- Inspect changed files or git diff enough to summarize implementation scope.
- Use read-only child Pi analysis.
- Write an advisory verification brief to the agreed artifact location.
- Include acceptance criteria checklist, scope review, changed files, verification evidence, and a verdict such as `pass`, `fail`, or `inconclusive`.

Out of scope:

- Completing the ticket.
- Committing changes.
- Moving files.
- Running destructive commands.
- Fixing failed verification automatically.

## Acceptance Criteria

- [ ] The command refuses to run when there is zero or more than one ongoing ticket.
- [ ] The verification brief includes an advisory notice and source ticket fingerprint.
- [ ] The brief references a plan artifact when one exists and warns if it is stale.
- [ ] The brief lists changed files or a changed-files snapshot.
- [ ] The brief marks unresolved acceptance criteria as failed or inconclusive, not silently passed.
- [ ] The brief recommends whether completion is ready for human decision without performing completion.

## Dependencies

- `PLF-020` — read-only child Pi runner.
- `PLF-022` — implementation plan artifacts are useful context for verification.

## Implementation Notes

Likely files:

- `.pi/extensions/ticket-workflow/`
- `.pi/extensions/ticket-workflow/prompts/verify.md` or equivalent inline prompt helper
- `tickets/.artifacts/verification/`
- `docs/ticket-workflow-commands.md`

The command may need deterministic parent-side git inspection. Keep it bounded and summarize rather than dumping large diffs into artifacts.

## Verification

- Run the command with no ongoing ticket and confirm it refuses clearly.
- Run the command with a controlled ongoing ticket and changed files.
- Inspect the verification brief for acceptance criteria coverage and scope warnings.
- Confirm no commit or ticket move occurs.
