# PLF-021: Add backlog ticket readiness brief command

## Objective

Add a read-only advisory command that analyzes whether a backlog ticket is ready to move to `planned/` and writes a readiness brief.

## Context

Backlog-to-planned refinement is one of the most repeated and context-heavy parts of the current workflow. A child Pi session can do the reading and analysis in a separate context, while the main orchestrator and human retain control over any actual ticket transition.

## Scope

In scope:

- Add a command such as `/ticket-readiness <ticket-id>`.
- Require the target ticket to exist in `tickets/backlog/`.
- Read the ticket workflow documentation and the target ticket.
- Use deterministic checks plus read-only child Pi analysis where useful.
- Write an advisory readiness brief to the agreed artifact location.
- Include a verdict such as `ready`, `not-ready`, `split-recommended`, or `defer-or-reject`.
- Identify missing objective, scope, acceptance criteria, dependencies, likely artifacts, and verification details.
- Include concrete human questions when the ticket is not ready.
- Hand the advisory result back to the parent LLM according to the `PLF-029` convention.

Out of scope:

- Moving the ticket to `planned/`.
- Editing the ticket.
- Creating split tickets automatically.
- Resolving dependencies automatically.

## Acceptance Criteria

- [ ] The command refuses tickets that are not currently in `tickets/backlog/`.
- [ ] The generated brief includes an advisory notice and source ticket fingerprint.
- [ ] The brief evaluates the workflow definition of ready.
- [ ] The brief lists missing information or blockers when the ticket is not ready.
- [ ] The brief recommends a next action without performing it.
- [ ] The command hands the readiness result back to the parent LLM or prints the suggested handoff in print mode.
- [ ] The command does not mutate ticket state.

## Dependencies

- `PLF-019` — shared advisory artifact conventions.
- `PLF-020` — read-only child Pi runner.
- `PLF-029` — parent LLM handoff convention for advisory command results.

## Implementation Notes

Likely files:

- `.pi/extensions/ticket-workflow/`
- `.pi/extensions/ticket-workflow/prompts/readiness.md` or equivalent inline prompt helper
- `tickets/.artifacts/readiness/`
- `docs/ticket-workflow-commands.md`

The prompt should ask the child to be useful but not authoritative. It should not tell the child to move files.

## Verification

- Run the command on an existing backlog ticket.
- Inspect the generated readiness brief.
- Change a copy or controlled ticket and confirm staleness detection behaves as designed.
- Confirm no ticket files move or change.
