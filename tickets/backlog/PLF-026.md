# PLF-026: Add ticket workflow doctor command

## Objective

Add a read-only audit command that checks ticket workflow consistency and reports actionable errors and warnings.

## Context

As ticket workflow commands and advisory artifacts accumulate, occasional consistency checks will help preserve the file-based workflow without adding hooks or a heavy project-management system.

## Scope

In scope:

- Add a command such as `/ticket-doctor`.
- Keep the command deterministic and display-only by default; do not trigger parent LLM handoff unless a later explicit option is approved.
- Detect duplicate ticket IDs across ticket state directories.
- Detect multiple ongoing tickets.
- Detect missing expected ticket directories.
- Detect malformed planned or ongoing tickets when required sections are absent.
- Detect stale advisory artifacts when artifact conventions support this.
- Detect artifacts whose source ticket no longer exists.
- Report errors and warnings separately.

Out of scope:

- Auto-repair.
- Moving tickets.
- Deleting stale artifacts.
- Git hooks.
- Replacing the manual workflow.

## Acceptance Criteria

- [ ] The command is read-only.
- [ ] It reports duplicate IDs and multiple ongoing tickets as errors.
- [ ] It reports missing required sections in planned or ongoing tickets as warnings or errors according to documented policy.
- [ ] It reports stale or orphaned advisory artifacts when detectable.
- [ ] It does not treat artifacts as authoritative workflow state.
- [ ] It provides actionable next steps without applying fixes.
- [ ] It does not trigger parent LLM handoff by default.

## Dependencies

- `PLF-019` — advisory artifact conventions are needed before stale artifact checks are meaningful.

Likely files:
