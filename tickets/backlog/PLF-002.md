# PLF-002: Parse Atom feeds into structured entries

Backlog tickets may be incomplete. Before moving this ticket to `planned/`, refine the scope and verification method.

## Objective

Add a small deterministic parser for already collected LinuxFr Atom feeds so agents can inspect feed entries as structured seed data.

## Context

`docs/linuxfr-v2-system-roadmap.md` identifies Atom feed parsing as part of the smallest coherent V2. The Agentic AI Career Radar prompts showed that feeds are useful seed sources, but raw XML inspection is awkward and encourages ad hoc LLM parsing.

Motivating sources:

- `docs/linuxfr-v2-system-roadmap.md`
- `docs/prototype-prompts/agentic-ai-career-radar-expanded/review.md`
- `docs/prototype-prompts/agentic-ai-career-radar-expanded/prompt.md`

## Scope

In scope:

- Parse local Atom files already stored under `data/raw/pages/`.
- Extract entry title, URL, published or updated date, author when available, categories/tags, and summary/snippet.
- Return enough information for agents to choose explicit detail URLs for later collection.

Out of scope:

- Fetching more feed pages automatically.
- Paginating feeds.
- Collecting detail pages automatically.
- Building a general XML processing platform.

## Acceptance Criteria

- [ ] To be refined before planning.
- [ ] Structured entries can be produced from at least one collected LinuxFr Atom feed.
- [ ] Output preserves source URL and local raw path.
- [ ] The parser remains local-only and bounded.

## Dependencies

## Implementation Notes

Likely implementation paths:

- Add an Atom-focused mode to `linuxfr_query_raw`; or
- Add a narrow helper used by candidate extraction/ranking features.

Keep it small and inspectable.
