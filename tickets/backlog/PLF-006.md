# PLF-006: Improve local raw query precision

Backlog tickets may be incomplete. Before moving this ticket to `planned`, refine the scope and verification method.

## Objective

Improve `linuxfr_query_raw` so local retrieval is less noisy and more useful for bounded research runs.

## Context

The Agentic AI Career Radar review noted noisy substring matches, weak term handling, and missing filters. The V2 roadmap calls for better raw query with multi-term search, type/date filters, snippets, and matched fields.

Motivating sources:

- `docs/linuxfr-v2-system-roadmap.md`
- `docs/prototype-prompts/agentic-ai-career-radar-expanded/review.md`

## Scope

In scope:

- Explore small query improvements such as multi-term matching, exact-word or normalized matching, type filters, URL filters, date filters when metadata exists, and better snippets.
- Return matched fields or match reasons when practical.
- Keep output compact enough for agent use.

Out of scope:

- Vector search.
- Semantic search.
- Full-text indexing service.
- Cross-run analytics.
