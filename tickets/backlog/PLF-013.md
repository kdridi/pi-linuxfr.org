# PLF-013: Define reusable report templates for cultural intelligence use cases

Backlog tickets may be incomplete. Before moving this ticket to `planned`, refine which use cases deserve templates first.

## Objective

Create lightweight Markdown templates for repeatable LinuxFr reports such as Contradiction Atlas, Weak Signal Radar, Scar Tissue Atlas, Prophecy Audit, Poll vs Praxis, and Sovereignty Ledger.

## Context

The use-case challenge prompts repeatedly proposed high-value report formats. The V2 roadmap also mentions topic report templates as a synthesis feature. Templates can improve repeatability without adding tools.

Motivating sources:

- `docs/linuxfr-v2-system-roadmap.md`
- `docs/use-case-challenge-prompts/bold-use-cases/`
- `docs/use-case-challenge-prompts/disruptive-cultural-intelligence/prompt.md`
- `docs/use-case-challenge-prompts/high-impact-user-prompts/prompt.md`

## Scope

In scope:

- Define a small set of report templates with required sections and citation expectations.
- Include source/corpus caveat sections to avoid overconfident synthesis.
- Keep templates compatible with the current collect/query/wiki loop.

Out of scope:

- Implementing a report-generation tool.
- Building a taxonomy or template engine.
- Creating polished product documentation for every possible use case.

## Acceptance Criteria

- [ ] To be refined before planning.
- [ ] At least two templates are concrete enough for a fresh agent to use.
- [ ] Templates require citations to source URLs or local raw paths.
- [ ] Templates include a coverage/limitations section.
