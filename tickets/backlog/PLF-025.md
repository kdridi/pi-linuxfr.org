# PLF-025: Add ongoing ticket completion brief command

## Objective

Add a helper command that prepares a completion checklist for the single ongoing ticket without committing or moving the ticket.

## Context

Ticket completion has several required checks: acceptance criteria, verification, changed files, decisions, commit message, and final ticket movement. A completion brief can make this reliable while keeping commit and state transition under human control.

## Scope

In scope:

- Add a command such as `/ticket-completion-brief`.
- Require exactly one ongoing ticket.
- Read the latest verification brief and warn if missing, stale, failed, or inconclusive.
- Summarize changed files.
- List missing completion requirements from `tickets/README.md`.
- Suggest a commit message using the ticket ID.
- Optionally write an advisory completion artifact if conventions include one.
- Hand the completion recommendation back to the parent LLM according to the `PLF-029` convention.

Out of scope:

- Running `git commit`.
- Moving the ticket to `completed/`.
- Editing the ticket resolution automatically.
- Deciding that completion is allowed without human approval.

## Acceptance Criteria

- [ ] The command refuses to run when there is zero or more than one ongoing ticket.
- [ ] The output includes changed files, verification status, and a suggested commit message.
- [ ] It warns when verification is missing, stale, failed, or inconclusive.
- [ ] It lists remaining completion checklist items.
- [ ] The command hands the completion recommendation back to the parent LLM or prints the suggested handoff in print mode.
- [ ] It does not commit or move files.

## Dependencies

- `PLF-023` — verification brief command.
- `PLF-029` — parent LLM handoff convention for advisory command results.

## Implementation Notes

This should be a practical checklist command, not an automatic finisher.

- Run with a controlled ongoing ticket and a verification artifact.
