# PLF-008: Parse collected comments into structured discussion data

Backlog tickets may be incomplete. Before moving this ticket to `planned`, refine the scope and verification method.

## Objective

Turn collected public LinuxFr comments into structured, queryable discussion evidence.

## Context

The V2 roadmap separates public comment collection from comment parsing. Comment parsing is needed before discussions can support coverage reports, contradiction maps, consensus summaries, and community reaction analysis.

Motivating sources:

- `docs/linuxfr-v2-system-roadmap.md`
- `docs/prototype-prompts/agentic-ai-career-radar-expanded/review.md`
- `docs/use-case-challenge-prompts/bold-use-cases/response-chatgpt.md`
- `docs/use-case-challenge-prompts/bold-use-cases/response-grok.md`

## Scope

In scope:

- Parse already collected public comment material.
- Preserve fields such as comment URL or anchor, author label when public, date when available, score when visible, parent/depth when available, text, and links.
- Keep enough context for cited discussion synthesis.

Out of scope:

- Collecting comments from the network.
- Inferring private information about authors.
- Sentiment scoring as a black box.
- Large-scale social graph analysis.

## Acceptance Criteria

- [ ] To be refined before planning.
- [ ] Parsed comments preserve source URL and local raw path.
- [ ] Thread/depth information is represented when available.
- [ ] The output can support later synthesis without replacing raw comments.
