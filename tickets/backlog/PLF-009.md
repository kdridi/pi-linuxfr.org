# PLF-009: Generate coverage reports for bounded corpora

Backlog tickets may be incomplete. Before moving this ticket to `planned`, refine the scope and verification method.

## Objective

Create a lightweight coverage report that tells agents what a bounded LinuxFr corpus covers well, weakly, or not at all.

## Context

The expanded radar review showed that the corpus was link-heavy, weak on RAG/evaluation/health-data-specific evidence, and missing comments. The V2 roadmap identifies coverage review as a quality stage before synthesis.

Motivating sources:

- `docs/linuxfr-v2-system-roadmap.md`
- `docs/prototype-prompts/agentic-ai-career-radar-expanded/review.md`

## Scope

In scope:

- Summarize source counts by type, topic/query term, date when available, and selected/candidate status when available.
- Highlight weak areas, overrepresented source types, failures, missing comments, and unsupported prompt dimensions.
- Produce an agent-readable Markdown or JSON report for one bounded run or selected source set.

Out of scope:

- Proving corpus representativeness for all LinuxFr.
- Automatic collection to fill gaps.
- Statistical modeling beyond simple counts and cited observations.

## Acceptance Criteria

- [ ] To be refined before planning.
- [ ] The report helps avoid overconfident synthesis from a weak corpus.
- [ ] The report cites local raw paths, manifest entries, or source URLs.
- [ ] The output can be reviewed before writing a final topic report.

## Dependencies

- None known. A run manifest would improve this but may not be strictly required for a manual first version.

## Implementation Notes

This could start as a prompt-only template before becoming a tool.
