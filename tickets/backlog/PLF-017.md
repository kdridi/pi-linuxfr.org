# PLF-017: Avoid sidebar and comment-noise candidates in HTML seed extraction

## Objective

Make `linuxfr_extract_candidates` prefer primary content candidates from HTML seed pages and avoid candidates whose anchor text and snippet come from navigation, sidebar, pagination, or latest-comment blocks.

## Context

`PLF-001` intentionally implemented broad local extraction from collected HTML. Post-reload testing showed that listing pages can produce candidates from non-primary page regions. These candidates may still canonicalize to valid detail URLs, but the context shown to the agent is misleading because it comes from comment/sidebar anchors rather than the listing item itself.

This cleanup should improve candidate usefulness without adding crawling or automatic collection.

## Reproduction

Run the tool against existing local listing pages:

```json
{
  "sourceType": "listing",
  "query": "entreprise",
  "limit": 3,
  "maxSources": 2
}
```

Observed result excerpt:

```text
LinuxFr candidate extraction: 74 unique local source(s), 2 selected, 2 readable (maxSources 2), 52 candidate occurrence(s), 3 returned (limit 3) for sourceType=listing, query=entreprise.
- score 20 link: https://linuxfr.org/users/vendrediouletrollsauvage/liens/la-chine-sanctionne-des-dizaines-d-entreprises-americaines — La Chine sanctionne des dizaines d’entreprises américaines
  source: data/raw/pages/listing-b5ba3a2825.html
  reason: classified as link; matched entreprise; has title or anchor text
  matched: entreprise
  excerpt: schema.org/Article"> Lien La
- score 8 journal: https://linuxfr.org/users/jch-2/journaux/galene-1-1 — Re: LiveKit
  source: data/raw/pages/listing-d4f8035667.html (2 sources)
  reason: classified as journal; found in 2 sources; has title or anchor text
  excerpt: Derniers commentaires Re: LiveKit <a href="/users/fernando-2
- score 8 journal: https://linuxfr.org/users/tkr/journaux/l-appli-gratuite-nouvelle-telecommande-obligatoire-du-pauvre — Re: Flash n'est pas vraiment mort
  source: data/raw/pages/listing-d4f8035667.html (2 sources)
  reason: classified as journal; found in 2 sources; has title or anchor text
  excerpt: portant#comment-2025666">Europe must choose between AI and climate goals, data center lobby says Re:
```

Problems:

- `Re: LiveKit` and `Re: Flash n'est pas vraiment mort` are comment/sidebar anchor texts, not primary listing item titles.
- The snippets include `Derniers commentaires` and broken HTML fragments.
- These candidates may be valid detail URLs, but the context is not the right reason to collect them.

Desired behavior: for HTML listing/tag pages, candidate extraction should prefer anchors from the main content area and candidate item blocks. Sidebar, navigation, pagination, login forms, feed links, and latest-comment links should either be ignored or strongly deprioritized with clear reasons.

## Scope

In scope:

- Filter or deprioritize HTML anchors from non-primary regions such as `aside#sidebar`, navigation, login forms, pagination, and latest-comment boxes.
- Prefer anchors inside main content containers and article/listing item blocks.
- Preserve valid candidates from real listing/tag result items.
- Add focused test scenarios for primary-content filtering.

Out of scope:

- Crawling or fetching new pages.
- Building a full DOM parser unless clearly simpler than targeted extraction.
- Semantic ranking beyond local structural filtering.
- Collecting comments.

## Acceptance Criteria

- [ ] The reproduction above no longer returns `Re: LiveKit` or `Re: Flash n'est pas vraiment mort` as top candidates due to sidebar/latest-comment context.
- [ ] Primary listing/tag item candidates are still extracted and classified.
- [ ] The tool remains deterministic, bounded, and local-only.
- [ ] The output reason or score remains inspectable.
- [ ] Unit or fixture scenarios cover the filtering cases below.

## Unit Test Scenarios

At minimum, cover these scenarios with helper-level tests or local fixture-based runtime checks:

1. **Primary article link is kept**
   - HTML contains a main content container with an `<article>` or item block linking to a LinuxFr detail URL.
   - Expected candidate is returned with the article title as `titleOrAnchor`.

2. **Sidebar latest-comment link is ignored or deprioritized**
   - HTML contains `aside#sidebar` or a latest-comments block with anchors such as `Re: LiveKit` pointing to `#comment-*` URLs.
   - Expected candidate is absent, or ranked below primary item candidates with a reason that makes the weaker context clear.

3. **Navigation and pagination links are ignored**
   - HTML contains `/news`, `/journaux`, pagination links, feed links, and login/account links.
   - Expected output does not include those as candidates unless `includeUnknown` explicitly allows unknown paths, and even then they should not be treated as detail candidates.

4. **Comment-fragment canonicalization does not hide bad context**
   - Input anchor points to `/users/example/journaux/title#comment-123` with anchor text `Re: unrelated subject` from a comment block.
   - Expected extraction does not promote it as if `Re: unrelated subject` were the journal title.

5. **Tag page main results still work**
   - Fixture resembles a LinuxFr tag page main results list.
   - Expected news, journal, link, forum, or poll detail URLs in the main results are extracted and classified.
